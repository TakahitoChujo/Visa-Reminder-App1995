## 1. テスト概要

### 1.1 目的

βテストの目的は以下の通りである。

1. **実ユーザー環境での品質検証**: 多様なデバイス・OS・言語設定での動作確認
2. **ユーザビリティ検証**: ターゲットユーザー（日本在住の外国人）が直感的に操作できるかの確認
3. **多言語対応の実用性検証**: 日本語・英語の翻訳品質と切替動作の確認
4. **通知機能の信頼性検証**: プッシュ通知がバックグラウンド含め正しく届くかの確認
5. **セキュリティの実用性検証**: AES-256-GCM暗号化がユーザー体験に影響しないかの確認
6. **ストア申請前の最終品質確認**: クラッシュ、パフォーマンス、UX上の重大問題の洗い出し

### 1.2 テスト期間

| フェーズ | 期間 | 内容 |
|---------|------|------|
| 準備期間 | 3月2日〜3月8日（1週間） | テスター募集、ビルド準備、配布環境セットアップ |
| テスト第1週 | 3月9日〜3月15日 | 基本機能テスト（登録・編集・チェックリスト） |
| テスト第2週 | 3月16日〜3月22日 | 通知テスト・エッジケース・総合評価 |
| 集計・分析期間 | 3月23日〜3月29日（1週間） | フィードバック集計、バグ修正、リリース判定 |

**合計**: 準備1週間 + テスト2週間 + 集計1週間 = **4週間**

### 1.3 テスター数の目標

| プラットフォーム | 目標人数 | 最低人数 |
|----------------|---------|---------|
| iOS | 15名 | 10名 |
| Android | 15名 | 10名 |
| **合計** | **30名** | **20名** |

**内訳の目安**:
- 日本語話者: 10名（日本人含む、在留資格制度に詳しい人）
- 英語話者: 10名（在日外国人、英語UIで検証）
- バイリンガル: 10名（言語切替機能の検証）

### 1.4 テスト環境

**iOS**:
- 対象OS: iOS 16.0以上
- 推奨デバイス: iPhone 12以降（SE第3世代含む）
- 配布: TestFlight

**Android**:
- 対象OS: Android 10（API 29）以上
- 推奨デバイス: Pixel 4以降、Galaxy S20以降、Xperia 5以降
- 配布: Google Play Internal Testing → Closed Testing

**ビルド環境**:
- Expo SDK: 54
- React Native: 0.81.5
- EAS Build使用（`eas build --platform all --profile preview`）

## 3. 配布方法

### 3.1 ビルド準備

**EAS Build設定**:

1. EAS CLIのインストールとログイン
```bash
npm install -g eas-cli
eas login
```

2. `eas.json` にpreviewプロファイルを追加
```json
{
  "build": {
    "preview": {
      "distribution": "internal",
      "ios": {
        "simulator": false,
        "buildConfiguration": "Release"
      },
      "android": {
        "buildType": "apk"
      },
      "env": {
        "APP_VARIANT": "beta"
      }
    }
  }
}
```

3. ビルド実行
```bash
eas build --platform all --profile preview
```

### 3.2 iOS配布（TestFlight）

**手順**:

1. **Apple Developer Account** でApp IDを登録（`com.residencereminder.app`）
2. **App Store Connect** でアプリを作成
3. EAS Buildで生成された`.ipa`をApp Store Connectにアップロード（`eas submit --platform ios`）
4. TestFlightでテスターグループ「Beta Testers」を作成
5. テスターのApple IDメールアドレスを追加（最大10,000名）
6. テスターに招待メールが自動送信される
7. テスターはTestFlightアプリ経由でインストール

**テスターへの案内事項**:
- App StoreからTestFlightアプリをインストールすること
- 招待メール内のリンクをタップしてβ版をインストール
- テスト版の有効期間は90日間

### 3.3 Android配布（Google Play Testing）

**手順**:

1. **Google Play Console** でアプリを作成（`com.residencereminder.app`）
2. EAS Buildで生成された`.aab`をアップロード
3. テストトラック設定:
   - **Internal Testing**（最初の1〜3日、開発チームで動作確認）
   - **Closed Testing**（テスター全員に配布）
4. Closed Testingでテスターリストを作成
5. テスターのGoogleアカウント（Gmailアドレス）を追加
6. テスターに参加リンクを共有

**テスターへの案内事項**:
- Gmailアドレスが必要
- 共有されたリンクからテストプログラムに参加
- Google Play Storeからインストール（「テスト版」表示あり）

### 3.4 配布チェックリスト

- [ ] EAS Build環境設定完了
- [ ] Apple Developer Account準備完了
- [ ] Google Play Console準備完了

## 4. テストケース

| ID | テスト内容 | 期待結果 | 優先度 |
|----|----------|---------|--------|
| BT-SETUP-001 | アプリを初回起動する | スプラッシュ画面表示後、ホーム画面が表示される | 必須 |
| BT-SETUP-002 | 通知許可ダイアログで「許可」を選択 | 通知権限が付与され、リマインダー機能が使用可能になる | 必須 |
| BT-SETUP-003 | 通知許可ダイアログで「許可しない」を選択 | アプリは使用可能だが、リマインダー設定画面に通知許可の案内が表示される | 必須 |
| BT-SETUP-004 | ホーム画面の空状態を確認 | 「登録がありません」のメッセージと「新規登録」ボタンが表示される | 必須 |

### 4.2 在留資格の登録

| ID | テスト内容 | 期待結果 | 優先度 |

## 5. フィードバック収集方法

### 5.1 アンケート（Google Forms）

**実施回数**: 2回
- **中間アンケート**: テスト第1週終了時（3月15日）
- **最終アンケート**: テスト第2週終了時（3月22日）

**中間アンケート項目**:

1. インストールはスムーズにできましたか？（5段階）
2. 在留資格の登録はスムーズにできましたか？（5段階）
3. 操作でわかりにくかった点はありますか？（自由記述）
4. クラッシュ（アプリの強制終了）は発生しましたか？（はい/いいえ → 発生した場合、状況を記述）
5. 表示が崩れていた画面はありますか？（自由記述）
6. 使用しているデバイスとOSバージョン（自由記述）

**最終アンケート項目**:

1. 全体的な使いやすさ（10段階: 0=非常に使いにくい〜10=非常に使いやすい）
2. 友人や知人にこのアプリを勧めますか？（NPS: 0〜10）
3. 各機能の満足度（5段階）:
   - 在留資格の登録
   - リマインダー通知
   - チェックリスト
   - メモ機能
   - 言語切替
4. 最も便利だと感じた機能（選択式 + 理由の自由記述）
5. 最も改善が必要だと感じた機能（選択式 + 理由の自由記述）
6. プレミアムプラン（¥600/年）に「こういう機能があれば課金する」と思う機能（自由記述）
7. アプリのデザイン・見た目の評価（5段階）
8. 日本語/英語の翻訳で不自然な箇所（自由記述）
9. 追加してほしい機能（自由記述）
10. その他自由なフィードバック（自由記述）

| 項目 | 合格基準 |

### 6.3 バグ重要度の定義

| 重要度 | 定義 | 例 | 対応期限 |
|--------|------|-----|---------|
| P0（致命的） | アプリが使用不能、データ損失 | クラッシュが頻発、登録データが消える、暗号化の不具合 | テスト期間中に修正しホットフィックス配布 |
| P1（重大） | 主要機能が使えない | 登録できない、通知が届かない、チェックが保存されない | リリース前に必ず修正 |
| P2（中度） | 機能は使えるが問題あり | レイアウト崩れ、翻訳の不自然さ、パフォーマンス低下 | リリースまでに修正を目指す |
| P3（軽微） | 使用に大きな支障なし | 微細なUIの不整合、誤字脱字 | リリース後パッチで対応可 |

| 致命的バグの発見 | 高 | ホットフィックスを48時間以内に配布、テスターに状況を即座に共有 |
| 通知がデバイスによって届かない | 中 | テスト第1週でプレビュー通知のテストを必須にし、早期に問題を発見 |
| App Store/Google Play審査落ち | 中 | βテストフィードバックで審査項目（プライバシー、クラッシュ等）を事前確認 |
| 翻訳の品質問題 | 低 | ネイティブ英語話者のテスターを優先的に確保し翻訳レビューを依頼 |
