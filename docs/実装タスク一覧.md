## 7. 実装順序の提案（MVP）

1. **基盤**: T1-1-* → T1-2-* → T1-3-*
2. **データ**: T2-1-* → T2-2-* → T2-3-*
3. **UI 基盤**: T4-1-*
4. **画面（登録→ホーム→他）**: T4-3-*（登録・編集）→ T4-2-*（ホーム）→ T4-4-*（リマインダー）→ T4-5-*（チェックリスト）→ T4-6-*（設定）
5. **通知**: T5-1 → T5-2 → T5-3 → T5-4
6. **品質・リリース**: T6-1, T6-2, T6-3 を並行または最後に実施。T6-4, T6-5 はリリース前

API・同期（T3-*）とサーバー側 DB（将来用）は MVP では省略可能。
