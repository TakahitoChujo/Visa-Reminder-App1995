# DateInput.web.tsx 改善提案

## 提案1: CSS min()関数を使用した柔軟な幅制限

### 現在のコード
```typescript
maxWidth: '320px', // 固定値
```

### 推奨コード
```typescript
maxWidth: 'min(320px, 100%)', // 画面幅に応じて自動調整
```

### 理由
- 画面幅が320px未満の場合でも、水平スクロールが発生しない
- CSS min()関数は、主要ブラウザ（Chrome 79+, Firefox 75+, Safari 11.1+, Edge 79+）でサポートされている
- より堅牢なレスポンシブデザインを実現

### 影響範囲
- 低リスク（既存の動作を改善するのみ）
- モバイル端末での使いやすさが向上

---

## 提案2: カスタムカレンダーピッカーライブラリの検討（長期的）

### 背景
HTML5の`<input type="date">`は、ブラウザによってカレンダーの表示位置や見た目が異なります。

### 検討候補
1. **react-datepicker**
   - カスタマイズ性が高い
   - ポップアップの位置を完全に制御可能
   - 約50KB（gzip後）

2. **react-day-picker**
   - 軽量（約15KB）
   - アクセシビリティに優れている
   - カスタマイズ可能

### メリット
- カレンダーの位置を完全に制御できる
- デザインの統一性が向上
- カスタム機能（範囲選択、無効日など）の追加が容易

### デメリット
- バンドルサイズが増加
- ブラウザネイティブのUIではなくなる
- 実装コストが増加

### 推奨
現時点では**不要**。maxWidth: 320pxの修正で十分対応できている。将来的にカスタム機能が必要になった場合に検討。

## 提案3: テストの自動化を優先

### 優先順位
1. **高**: ユニットテストの実行環境を整備（@testing-library/react のインストール）
2. **中**: CI/CDパイプラインへのテスト組み込み
3. **低**: E2Eテストの実行（Playwrightのセットアップ）

### 実装手順
```bash
# ステップ1: ユニットテストライブラリのインストール
npm install --save-dev @testing-library/react @testing-library/jest-dom

# ステップ2: テスト実行
npm test

# ステップ3: カバレッジレポートの生成
npm test -- --coverage
```
