# 要件定義書：在留資格更新リマインダー＋手続きガイド アプリ

**文書バージョン**: 1.0  
**作成日**: 2025年2月14日  
**対象企画**: 企画.md ④ 在留資格更新リマインダー＋手続きガイド（最推し）

---

## 1. プロジェクト概要

### 1.1 アプリ名（仮）
在留資格更新リマインダー（略称案：ビザリマインド / 在留リマ等）

### 1.2 目的
在留資格（ビザ）の有効期限更新を忘れずに行えるよう、**期限リマインダー**と**資格別の必要書類・手続きチェックリスト**を提供する個人向けアプリ。更新忘れによる在留資格失効リスクを軽減する。

### 1.3 背景・課題
- 在留資格更新は**有効期限の約4ヶ月前〜**から申請可能。忘れると失効に直結する。
- 資格タイプごとに必要書類・申請時期が異なり、個人で把握しづらい。
- 「在留資格更新リマインダー」に特化したアプリは現状ほぼ存在しない（競合優位性あり）。
- 在留外国人は約358万人（2024年）。就労・配偶者等・永住者配偶者等など更新ありの資格が多数。

### 1.4 スコープ
- **対象**: 個人ユーザー（法人向け入管ソフトとは別軸）。
- **プラットフォーム**: まずは **iOS / Android ネイティブ or クロスプラットフォーム（React Native / Flutter 等）** を想定。Web は将来オプション。
- **言語**: MVP は日本語必須。余力があれば多言語（英語・ベトナム語・中国語等）。

## 2. ユーザー（ペルソナ）

| 項目 | 内容 |
|------|------|
| **主ペルソナ** | 日本で就労している外国人（技術・人文・技能等）。在留カードの有効期限を把握し、更新時期に忘れず手続きしたい。 |
| **副ペルソナ** | 日本人・永住者の配偶者等。更新時期（有効期限の約4ヶ月前〜）を忘れがちで、必要書類をまとめて把握したい。 |
| **ニーズ** | ・期限の「見える化」とリマインド ・資格別の必要書類・手続きの一元把握 ・シンプルで信頼できるUI |

## 3. 機能要件

### 3.1 必須機能（MVP）

#### F1. 在留カード・資格の登録
| 項目 | 内容 |
|------|------|
| 概要 | ユーザーが自分の在留資格情報を登録する。 |
| 詳細 | ・**有効期限（年月日）** の登録（必須） ・**資格タイプ** の選択（就労系・配偶者等・永住申請前など。一覧は事前定義） ・1ユーザーあたり **1件のメイン登録** を想定（家族分は将来拡張で検討） ・登録日・更新日の記録（履歴用） |
| 受入条件 | ・有効期限のみでも登録可能（資格タイプは「その他」で可） ・資格タイプ選択時、その資格の「申請可能時期」の目安を表示できる（例：有効期限の4ヶ月前〜） |

#### F2. リマインダー
| 項目 | 内容 |
|------|------|
| 概要 | 在留資格の有効期限に基づき、事前に通知する。 |
| 詳細 | ・**通知タイミング**：有効期限の 4ヶ月前 / 3ヶ月前 / 1ヶ月前 / 2週間前（ユーザーがオン/オフ可能） ・**プッシュ通知** および **アプリ内バッジ・一覧での表示** ・申請可能時期の開始日（例：4ヶ月前の日付）を明確に表示 ・リマインダー設定のカスタム（例：6ヶ月前を追加）は MVP では固定でよい |
| 受入条件 | ・OS の通知許可を取得し、指定日にプッシュ送信できる ・アプリ未起動でも通知が届く ・通知タップでアプリ起動し、該当カード詳細またはチェックリストに遷移できる |

#### F3. 資格別「必要書類・手続き」チェックリスト
| 項目 | 内容 |
|------|------|
| 概要 | 資格タイプに応じた必要書類・手順をチェックリスト形式で表示し、進捗を記録できる。 |
| 詳細 | ・**パターン**：就労系（技術・人文・技能等）／配偶者等（日本人配偶者等・永住者配偶者等）／その他（永住申請前の準備等） ・各パターンに **項目**（例：在留カード、写真、証明書類、申請書類名など）を事前定義 ・ユーザーは各項目を「未着手／準備中／完了」などでチェック可能（シンプルにチェックのみでも可） ・内容は法改正に伴い更新する前提のため、**管理画面または設定で項目を編集可能**にしておくか、初版は静的リストでよい ・「入管のURL」「申請窓口の案内」などのリンクを載せられるようにする |
| 受入条件 | ・資格タイプを選択した登録に対して、対応するチェックリストが表示される ・チェック状態は端末内に保存（アカウント連携時はサーバ保存） |

### 3.2 推奨機能（MVP 後）

| ID | 機能概要 | 優先度 |
|----|----------|--------|
| F4 | 多言語対応（日本語・英語・ベトナム語・中国語等） | 高 |
| F5 | 家族・複数資格の登録（2件目以降は有料オプション案） | 中 |
| F6 | 申請完了・次回更新日の登録（更新後の新しい有効期限で再リマインド） | 高 |
| F7 | 過去のリマインダー・申請履歴の表示 | 中 |
| F8 | CSV/PDF でのチェックリスト出力（印刷・行政書士提出用） | 低 |

### 3.3 非機能要件（MVP で考慮）

| 項目 | 内容 |
|------|------|
| **パフォーマンス** | 一覧・詳細の表示は 2 秒以内を目安。 |
| **セキュリティ** | 在留カード番号や氏名などは保存しない設計を推奨（有効期限と資格タイプのみ）。保存する場合は暗号化。 |
| **プライバシー** | 利用規約・プライバシーポリシーを用意し、データの取得範囲・利用目的を明示。 |
| **可用性** | オフラインでも登録済みの期限・チェックリストの閲覧・編集が可能であることが望ましい（同期はオンライン時）。 |

## 4. 収益・課金要件

### 4.1 課金モデル案
- **月額**: 300〜500円（継続課金）
- **年額**: 1,200〜1,500円（更新は1〜3年ごとのため年額も成立）
- **月1万の目安**: 月額400円 × 25人 または 年額1,500円で月7〜8本

### 4.2 無料／有料の線引き（案）
- **無料**: 在留資格1件の登録、リマインダー（4・3・1ヶ月前・2週間前）、チェックリストの閲覧・チェックまで。
- **有料**: ・複数資格（家族分）の登録 または ・過去履歴の長期保持 または ・PDF/CSV 出力 など。  
※ 初版 MVP では「無料で上記すべて提供し、複数登録 or 多言語 or 出力機能で有料」などシンプルな線引きも可。

### 4.3 課金基盤
- ストア課金（App Store / Google Play）を想定。サブスク・年額の両方に対応できるようにする。

## 5. 画面・UI 要件（概要）

| 画面 | 概要 |
|------|------|
| **ホーム／一覧** | 登録中の在留資格の有効期限、次回リマインド日、「残り○日」を一覧表示。チェックリストへのショートカット。 |
| **登録・編集** | 有効期限・資格タイプの入力・変更。 |
| **リマインダー設定** | 通知のオン/オフ、通知タイミングの表示（MVP は固定で可）。 |
| **チェックリスト** | 資格タイプに応じた項目一覧。チェック ON/OFF。必要に応じてメモ欄。 |
| **設定** | 通知許可、言語（多言語対応時）、課金状態、利用規約・プライバシーポリシー、お問い合わせ。 |

※ ワイヤーフレーム・デザインは別途作成。

## 6. データ要件（概念）

### 6.1 ユーザーが登録するデータ（例）
- 在留資格の有効期限（日付）
- 資格タイプ（コードまたはマスタID）
- リマインダー ON/OFF 及び通知タイミング（固定の場合は設定不要）
- チェックリストの各項目の完了状態（およびメモがあればメモ）

### 6.2 保持しないデータ（推奨）
- 在留カード番号、氏名、生年月日、住所など個人を特定し得る機微情報（取得しない設計を推奨）。

### 6.3 マスタデータ
- 資格タイプ一覧（表示名、申請可能時期の目安、紐づくチェックリストID）
- チェックリストテンプレート（資格タイプ別の項目一覧）

## 7. 外部連携・制約

| 項目 | 内容 |
|------|------|
| **入管・法務省** | 公式APIは利用しない。必要書類・手続きは公式サイトの情報を参照し、アプリ内では「参考ガイド」として記載。法改正時はコンテンツ更新が必要。 |
| **通知** | FCM（Android）/ APNs（iOS）を用いたプッシュ通知。 |
| **課金** | App Store / Google Play のストア課金。 |
